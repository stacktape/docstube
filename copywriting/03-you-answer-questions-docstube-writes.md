## Why docstube

- **You don't write the docs.** A writer agent does, from your real source. You review and steer.
- **Verified, not hallucinated.** Every factual claim is grounded in the code and machine-checked before it ships.
- **Always-current.** Change the code and only the affected pages regenerate — locally or as a GitHub Action PR.
- **Written for your readers.** Name your personas; each page is reviewed for the right depth and tone.
- **Bring your own AI.** Uses your existing Claude, Codex, or Gemini subscription, or any compatible API key, with spend caps you set.
- **Complete.** Coverage is planned from a codemap, so gaps get surfaced instead of silently skipped.
- **Open source.** MIT, generates MDX, no lock-in.

## How it works

1. Point docstube at your repo. A local web UI wizard opens.
2. Choose the doc type, define reader personas, add a little context.
3. A writer agent reads your code; reviewer and verifier agents check it for audience fit and accuracy.
4. Review the result rendered exactly as it'll look live, and leave feedback on any element.
5. The writer agent refines until you're happy.
6. On every code change, only the affected pages regenerate.

After step 2, you can walk away. The rest runs without you.

## The gate — why hands-off is safe

Letting it run doesn't mean trusting it blindly. It means trusting the checks every page has to clear first.

- If a code sample doesn't compile, the page doesn't ship. (`tsc` for TypeScript, `pyright` for Python.)
- Imports resolve against the real package — phantom functions never reach the reader.
- Broken links and invalid MDX are machine-checked. They don't reach readers.
- The agent that writes the page isn't the agent that judges it.
- No page ships with an open blocker. That's not a policy — it's the build.

That's what lets you stop supervising: the gate is doing the supervising.

## FAQ

**Do I really not write anything?** Right. You define the doc type and audience; the writer agent drafts and refines. You review the rendered result and steer where you want.

**How do I know it's not making things up?** Every page clears deterministic checks and cross-vendor review before it ships. Samples that don't compile and links that don't resolve never reach the page.

**Does my code leave my machine?** Source context goes only to the AI provider you already use, on your own credentials — never to docstube's servers.

**What if I want to change something?** Leave feedback on any element in the wizard; the writer agent refines until you're happy.
